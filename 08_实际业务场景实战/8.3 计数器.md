## 8.3 计数器

### 场景故事：实时数据统计的需求

**故事背景**：
某新闻网站需要实时统计文章的阅读量，以便向用户展示热门文章。技术团队最初使用数据库计数，但随着访问量的增加，频繁的数据库更新导致性能下降，统计数据出现延迟。同时，网站还需要限制API调用频率，防止恶意请求。

**业务需求**：
1. 实时统计文章阅读量，确保数据及时更新
2. 限制API调用频率，防止滥用
3. 支持高并发场景，应对突发流量
4. 数据持久化，避免服务器重启后数据丢失
5. 支持按时间维度统计（日、周、月）

### 应用场景

- 网站访问量统计：实时统计页面浏览次数
- 商品库存管理：实时更新库存数量
- 接口调用次数限制：防止API滥用
- 实时数据统计：实时监控系统指标

### 方案对比

**方案一：基于数据库的计数器**
- **优点**：
  - 数据持久化，可靠性高
  - 支持复杂查询和分析
  - 事务支持，数据一致性好
- **缺点**：
  - 性能差，频繁更新导致数据库压力大
  - 并发能力有限，容易产生锁竞争
  - 实时性差，统计数据有延迟

**方案二：基于内存的计数器**
- **优点**：
  - 性能极高，内存操作
  - 实时性好，无延迟
- **缺点**：
  - 数据易丢失，服务器重启后数据清零
  - 不支持分布式环境
  - 内存容量有限

**方案三：基于Redis的计数器**
- **优点**：
  - 高性能，支持高并发
  - 原子操作，数据一致性好
  - 支持过期时间，自动清理
  - 数据持久化选项（RDB/AOF）
  - 支持分布式部署
  - 丰富的命令集，支持多种计数场景
- **缺点**：
  - 需要额外的Redis服务器资源
  - 网络传输开销
  - 数据可能存在短暂不一致（在主从复制场景下）

**选择理由**：
考虑到新闻网站的实时统计需求和高并发场景，选择基于Redis的计数器方案。Redis的原子操作能够保证计数的准确性，高性能特性能够应对突发流量，同时数据持久化选项确保数据安全性。相比其他方案，Redis计数器在性能、可靠性和功能丰富度之间取得了最佳平衡。

### 计数器原理

**核心原理**：利用Redis的原子操作INCR/DECR来实现计数器

**优势**：
- 高性能：Redis单线程模型，原子操作
- 实时性：实时更新，实时读取
- 可靠性：持久化保证数据不丢失
- 灵活性：支持多种计数场景

### 计数器类型

1. **简单计数器**：基本的增减操作
2. **限时计数器**：在特定时间窗口内计数
3. **分布式计数器**：在分布式环境下的计数
4. **排行榜计数器**：带排序的计数器

### 代码实现

```go
package main

import (
	"context"
	"fmt"
	"time"

	"github.com/go-redis/redis/v8"
)

// CounterService 计数器服务
type CounterService struct {
	rdb *redis.Client
}

// NewCounterService 创建计数器服务
func NewCounterService(rdb *redis.Client) *CounterService {
	return &CounterService{rdb: rdb}
}

// Increment 增加计数
func (s *CounterService) Increment(ctx context.Context, key string) (int64, error) {
	return s.rdb.Incr(ctx, key).Result()
}

// Decrement 减少计数
func (s *CounterService) Decrement(ctx context.Context, key string) (int64, error) {
	return s.rdb.Decr(ctx, key).Result()
}

// GetCount 获取计数值
func (s *CounterService) GetCount(ctx context.Context, key string) (int64, error) {
	return s.rdb.Get(ctx, key).Int64()
}

// SetCount 设置计数值
func (s *CounterService) SetCount(ctx context.Context, key string, value int64) error {
	return s.rdb.Set(ctx, key, value, 0).Err()
}

// IncrementWithExpire 带过期时间的增加计数
func (s *CounterService) IncrementWithExpire(ctx context.Context, key string, expiration time.Duration) (int64, error) {
	// 先增加计数
	count, err := s.rdb.Incr(ctx, key).Result()
	if err != nil {
		return 0, err
	}

	// 如果是第一次计数，设置过期时间
	if count == 1 {
		err = s.rdb.Expire(ctx, key, expiration).Err()
		if err != nil {
			return count, err
		}
	}

	return count, nil
}

// BatchIncrement 批量增加计数
func (s *CounterService) BatchIncrement(ctx context.Context, keys []string) (map[string]int64, error) {
	pipe := s.rdb.Pipeline()
	cmds := make(map[string]*redis.IntCmd)

	for _, key := range keys {
		cmds[key] = pipe.Incr(ctx, key)
	}

	_, err := pipe.Exec(ctx)
	if err != nil {
		return nil, err
	}

	result := make(map[string]int64)
	for key, cmd := range cmds {
		result[key], _ = cmd.Result()
	}

	return result, nil
}

// 使用示例
func main() {
	ctx := context.Background()

	// 创建Redis客户端
	rdb := redis.NewClient(&redis.Options{
		Addr: "localhost:6379",
	})

	// 创建计数器服务
	counterService := NewCounterService(rdb)

	// 示例1：简单计数
	fmt.Println("=== 简单计数 ===")
	for i := 0; i < 5; i++ {
		count, err := counterService.Increment(ctx, "counter:page:view")
		if err != nil {
			fmt.Printf("增加计数失败: %v\n", err)
			continue
		}
		fmt.Printf("页面访问次数: %d\n", count)
	}

	// 示例2：限时计数
	fmt.Println("\n=== 限时计数 ===")
	for i := 0; i < 3; i++ {
		count, err := counterService.IncrementWithExpire(ctx, "counter:api:call", 1*time.Minute)
		if err != nil {
			fmt.Printf("增加计数失败: %v\n", err)
			continue
		}
		fmt.Printf("API调用次数: %d\n", count)
	}

	// 示例3：批量计数
	fmt.Println("\n=== 批量计数 ===")
	keys := []string{"counter:user:login", "counter:order:create", "counter:product:view"}
	results, err := counterService.BatchIncrement(ctx, keys)
	if err != nil {
		fmt.Printf("批量增加计数失败: %v\n", err)
		return
	}

	for key, count := range results {
		fmt.Printf("%s: %d\n", key, count)
	}

	// 获取最终计数
	finalCount, err := counterService.GetCount(ctx, "counter:page:view")
	if err != nil {
		fmt.Printf("获取计数失败: %v\n", err)
		return
	}
	fmt.Printf("\n最终页面访问次数: %d\n", finalCount)
}
```

### 计数器的应用场景

#### 网站访问统计
- **日访问量**：`counter:visit:20240101`
- **小时访问量**：`counter:visit:20240101:12`
- **实时访问量**：`counter:visit:realtime`

#### 接口限流
- **IP限流**：`counter:rate:ip:192.168.1.100`
- **用户限流**：`counter:rate:user:12345`
- **接口限流**：`counter:rate:api:/v1/users`

#### 库存管理
- **商品库存**：`counter:stock:product:123`
- **库存预警**：当库存低于阈值时触发预警

### 计数器的性能优化

1. **批量操作**：使用Pipeline减少网络往返
2. **合理设置过期时间**：避免内存占用过高
3. **监控计数趋势**：及时发现异常情况
4. **使用Redis集群**：提高并发处理能力

### 最佳实践

1. **键名设计**：使用业务前缀+时间维度
2. **过期策略**：根据业务需求设置合理的过期时间
3. **错误处理**：妥善处理计数操作的错误
4. **监控告警**：设置计数异常的监控和告警
5. **数据持久化**：确保计数数据的可靠性
