## 8.4 排行榜

### 场景故事：游戏排行榜的实时更新需求

**故事背景**：
某手机游戏平台需要实现一个实时排行榜功能，展示玩家的游戏分数排名。随着用户量的增长，传统的数据库排序方式已无法满足实时性要求，每次更新排行榜都需要全表扫描和排序，导致系统响应缓慢，用户体验差。

**业务需求**：
1. 实时更新排行榜，玩家分数变化后立即反映在排行榜上
2. 支持多种排行榜类型（日榜、周榜、月榜、总榜）
3. 支持查看特定玩家的排名和分数
4. 支持前N名玩家的快速查询
5. 系统需要支持高并发，应对大量玩家同时更新分数的场景

### 应用场景

- 游戏排行榜：实时展示玩家分数排名
- 销售排行榜：展示商品销售情况
- 人气排行榜：展示内容或用户的人气值
- 积分排行榜：展示用户积分排名

### 方案对比

**方案一：基于数据库的排行榜**
- **优点**：
  - 数据持久化，可靠性高
  - 支持复杂查询和分析
  - 事务支持，数据一致性好
- **缺点**：
  - 性能差，排序操作开销大
  - 实时性差，需要定期计算
  - 并发能力有限，容易产生锁竞争
  - 难以支持多种时间维度的排行榜

**方案二：基于内存的排行榜**
- **优点**：
  - 性能高，内存操作
  - 实时性好，无延迟
- **缺点**：
  - 数据易丢失，服务器重启后数据清零
  - 不支持分布式环境
  - 内存容量有限，难以存储大量数据

**方案三：基于Redis的排行榜**
- **优点**：
  - 高性能，支持实时更新
  - 内置排序功能，查询速度快
  - 支持多种时间维度的排行榜
  - 数据持久化选项（RDB/AOF）
  - 支持分布式部署
  - 丰富的命令集，支持排名查询、分数查询等操作
- **缺点**：
  - 需要额外的Redis服务器资源
  - 网络传输开销
  - 数据可能存在短暂不一致（在主从复制场景下）
  - 内存使用量较大，需要合理规划

**选择理由**：
考虑到游戏平台的实时性要求和高并发场景，选择基于Redis的排行榜方案。Redis的Sorted Set数据结构天生适合实现排行榜功能，支持快速的插入、更新和查询操作。相比其他方案，Redis排行榜在性能、实时性和功能丰富度方面都具有明显优势，能够满足游戏平台的业务需求。

### 排行榜原理

**核心原理**：利用Redis的Sorted Set数据结构实现排行榜

**Sorted Set特性**：
- 有序性：按分数排序
- 唯一性：每个成员唯一
- 高效性：插入、删除、查询都是O(logN)
- 灵活性：支持正序和倒序排列

### 代码实现

```go
package main

import (
	"context"
	"fmt"

	"github.com/go-redis/redis/v8"
)

// LeaderboardService 排行榜服务
type LeaderboardService struct {
	rdb *redis.Client
}

// NewLeaderboardService 创建排行榜服务
func NewLeaderboardService(rdb *redis.Client) *LeaderboardService {
	return &LeaderboardService{rdb: rdb}
}

// AddScore 添加分数
func (s *LeaderboardService) AddScore(ctx context.Context, leaderboard string, member string, score float64) error {
	return s.rdb.ZAdd(ctx, leaderboard, &redis.Z{
		Score:  score,
		Member: member,
	}).Err()
}

// GetRank 获取排名（从0开始）
func (s *LeaderboardService) GetRank(ctx context.Context, leaderboard string, member string) (int64, error) {
	return s.rdb.ZRevRank(ctx, leaderboard, member).Result()
}

// GetScore 获取分数
func (s *LeaderboardService) GetScore(ctx context.Context, leaderboard string, member string) (float64, error) {
	return s.rdb.ZScore(ctx, leaderboard, member).Result()
}

// GetTopN 获取前N名
func (s *LeaderboardService) GetTopN(ctx context.Context, leaderboard string, n int64) ([]redis.Z, error) {
	return s.rdb.ZRevRangeWithScores(ctx, leaderboard, 0, n-1).Result()
}

// GetRange 获取指定范围的排名
func (s *LeaderboardService) GetRange(ctx context.Context, leaderboard string, start, stop int64) ([]redis.Z, error) {
	return s.rdb.ZRevRangeWithScores(ctx, leaderboard, start, stop).Result()
}

// GetRankWithScore 获取排名和分数
func (s *LeaderboardService) GetRankWithScore(ctx context.Context, leaderboard string, member string) (int64, float64, error) {
	// 使用Pipeline批量操作
	pipe := s.rdb.Pipeline()
	rankCmd := pipe.ZRevRank(ctx, leaderboard, member)
	scoreCmd := pipe.ZScore(ctx, leaderboard, member)
	_, err := pipe.Exec(ctx)
	if err != nil {
		return 0, 0, err
	}

	rank, err := rankCmd.Result()
	if err != nil {
		return 0, 0, err
	}

	score, err := scoreCmd.Result()
	if err != nil {
		return 0, 0, err
	}

	return rank, score, nil
}

// RemoveMember 移除成员
func (s *LeaderboardService) RemoveMember(ctx context.Context, leaderboard string, member string) error {
	return s.rdb.ZRem(ctx, leaderboard, member).Err()
}

// GetCount 获取成员数量
func (s *LeaderboardService) GetCount(ctx context.Context, leaderboard string) (int64, error) {
	return s.rdb.ZCard(ctx, leaderboard).Result()
}

// 使用示例
func main() {
	ctx := context.Background()

	// 创建Redis客户端
	rdb := redis.NewClient(&redis.Options{
		Addr: "localhost:6379",
	})

	// 创建排行榜服务
	leaderboardService := NewLeaderboardService(rdb)

	// 排行榜名称
	leaderboard := "leaderboard:game:score"

	// 添加分数
	players := map[string]float64{
		"player1": 1000,
		"player2": 1500,
		"player3": 900,
		"player4": 2000,
		"player5": 1200,
	}

	for player, score := range players {
		err := leaderboardService.AddScore(ctx, leaderboard, player, score)
		if err != nil {
			fmt.Printf("添加分数失败 %s: %v\n", player, err)
		}
	}

	// 获取前3名
	fmt.Println("=== 排行榜前3名 ===")
	top3, err := leaderboardService.GetTopN(ctx, leaderboard, 3)
	if err != nil {
		fmt.Printf("获取排行榜失败: %v\n", err)
		return
	}

	for i, item := range top3 {
		fmt.Printf("第%d名: %s, 分数: %.0f\n", i+1, item.Member, item.Score)
	}

	// 获取指定玩家的排名和分数
	player := "player3"
	rank, score, err := leaderboardService.GetRankWithScore(ctx, leaderboard, player)
	if err != nil {
		fmt.Printf("获取玩家排名失败: %v\n", err)
		return
	}
	fmt.Printf("\n玩家 %s 的排名: 第%d名, 分数: %.0f\n", player, rank+1, score)

	// 更新玩家分数
	fmt.Println("\n=== 更新玩家分数 ===")
	err = leaderboardService.AddScore(ctx, leaderboard, "player3", 1800)
	if err != nil {
		fmt.Printf("更新分数失败: %v\n", err)
		return
	}

	// 再次获取玩家排名
	rank, score, err = leaderboardService.GetRankWithScore(ctx, leaderboard, player)
	if err != nil {
		fmt.Printf("获取玩家排名失败: %v\n", err)
		return
	}
	fmt.Printf("玩家 %s 的新排名: 第%d名, 新分数: %.0f\n", player, rank+1, score)

	// 获取排行榜总人数
	count, err := leaderboardService.GetCount(ctx, leaderboard)
	if err != nil {
		fmt.Printf("获取排行榜人数失败: %v\n", err)
		return
	}
	fmt.Printf("\n排行榜总人数: %d\n", count)
}
```

### 排行榜的高级功能

#### 时间维度排行榜
- **日榜**：`leaderboard:game:score:20240101`
- **周榜**：`leaderboard:game:score:2024W01`
- **月榜**：`leaderboard:game:score:202401`
- **总榜**：`leaderboard:game:score:total`

#### 多维度排行榜
- **分数排行榜**：按分数排序
- **等级排行榜**：按等级排序
- **活跃度排行榜**：按活跃度排序

#### 排行榜缓存
- **本地缓存**：减少Redis访问
- **批量查询**：减少网络往返
- **预计算**：定期计算排行榜数据

### 排行榜的性能优化

1. **合理设置过期时间**：避免内存占用过高
2. **使用Pipeline**：批量操作减少网络开销
3. **限制排行榜大小**：只保留前N名
4. **定期清理**：清理过期的排行榜数据

### 最佳实践

1. **键名设计**：使用业务前缀+时间维度
2. **分数计算**：根据业务需求设计合理的分数计算规则
3. **更新频率**：根据业务需求设置合适的更新频率
4. **监控告警**：监控排行榜的性能和数据完整性
5. **数据备份**：定期备份排行榜数据
